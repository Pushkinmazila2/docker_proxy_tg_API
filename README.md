# docker_proxy_tg_API v2.0

Telegram API Proxy Service (Версия 2.0) для Docker и CI/CD

## Описание

Данный проект предоставляет代理服务 для Telegram API, настроенный через Go и Nginx. Приложение работает как прокси-сервер, обрабатывающий запросы к Telegram Bot API.

## Структура проекта

| Файл | Назначение |
|------|------------|
| `main.go` | Основной код приложения (Go) |
| `nginx.conf` | Конфигурация Nginx для проксирование запросов |
| `Dockerfile` | Контейнеризация Go-приложения |
| `docker-compose.yml` | Компоцирование Docker-контейнеров |

## Работа с секретами

Приложение использует секреты через CI/CD платформу (GitLab, GitHub Actions и т.д.). Секретные переменные определяются следующим образом:

1. **Secret Variable** - `MYSECRET_WAY`
   - Используется для аутентификации доступа к Telegram API
   - Хранится в настройках CI/CD системы
   - Доступный через окружение в контейнере при запуске

2. **Важно:** Не храните пароли или секретные ключи в репозитории Git! Секреты должны быть указаны только в CI/CD системах и не передаваться через публичные файлы.

## Установка и запуск

### Локальный запуск

```bash
# Клонировать репозиторий
git clone <repository-url>
cd docker_proxy_tg_API

# Запустить контейнер с помощью Docker Compose
docker-compose up -d
```

### Сборка и запуск приложения

```bash
# Собрать Go приложение
go build -o proxy .

# Запустить прокси-сервер (порт по умолчанию: 8080)
PORT=8080 ./proxy
```

## Конфигурация секрета

Примечание: Секретная конфигурация для Telegram API не включается в репозиторий. Для работы проекта необходимо настроить CI/CD систему с переменной `MYSECRET_WAY`, содержащей данные аутентификации Telegram API.

Убедитесь, что переменная доступна при запуске Docker-контейнера через:
```yaml
# docker-compose.yml (пример)
environment:
  - MYSECRET_WAY=<ВАШ_СКРЕТ>
```
